Primero, se ingresa al portal inmobiliario para aplicar los filtros necesarios, seleccionando la ubicación en Capital Federal y la condición de propiedades usadas. 
Debido a que el sitio cuenta con medidas de seguridad que dificultan el acceso automatizado directo, se utiliza un script de JavaScript que se ejecuta desde la consola del navegador. 
Este procedimiento permite descargar archivos CSV con los enlaces, títulos y precios iniciales de las publicaciones, los cuales se almacenan en una carpeta de datos crudos.
Luego, se utiliza un script de Python que consolida todos los archivos descargados y elimina los registros duplicados basándose en el enlace de la publicación. 
Una vez obtenida la lista única, el código recorre cada enlace de forma individual, aplicando tiempos de espera entre cada petición para evitar bloqueos. 
Durante esta navegación, el script captura la descripción completa del aviso y la tabla de especificaciones técnicas, obteniendo información detallada que no está disponible en el listado general.
Con toda la información recolectada, se ejecuta un proceso de limpieza y minería de datos mediante expresiones regulares (RegEx). 
Este análisis permite escanear las descripciones largas para extraer datos que a veces no están estructurados, como la cantidad de dormitorios, baños y la presencia de servicios adicionales 
como cochera, balcón o terraza. También se identifican términos específicos como "a refaccionar", "reciclar" o "estado original" para clasificar automáticamente las propiedades que representan una oportunidad de inversión para remodelación y reventa.
En la etapa final, el proceso se encarga de normalizar los valores numéricos y corregir los errores de formato en los precios. El script transforma las cadenas de texto en números 
enteros reales, asegurando que los valores de mercado sean precisos y eliminando artefactos visuales como decimales innecesarios. Con los precios y las superficies ya normalizados, se calcula el precio por metro cuadrado para cada propiedad. 
El resultado de este flujo de trabajo es un dataset final consolidado, limpio y con indicadores clave, listo para ser utilizado en el análisis estadístico del proyecto.

let casas = [];
// Buscamos en los dos tipos de contenedores posibles
document.querySelectorAll('li.ui-search-layout__item, .ui-search-result__wrapper').forEach(item => {
    // Precio (esta funcionaba bien)
    let precio = item.querySelector('.andes-money-amount__fraction')?.innerText || "N/A";
    
    // Título (buscamos por etiqueta h2 o por clase nueva)
    let titulo = item.querySelector('h2, .poly-component__title')?.innerText || "N/A";
    
    // Ubicación (clase vieja o clase nueva)
    let loc = item.querySelector('.ui-search-item__location, .poly-component__location')?.innerText || "N/A";
    
    // Link (el primer enlace que encuentre)
    let link = item.querySelector('a')?.href || "N/A";
    
    // Atributos (m2, ambientes, etc)
    let attrs = Array.from(item.querySelectorAll('.ui-search-card-attributes__attribute, .poly-attributes-list__item')).map(a => a.innerText).join(' | ');

    casas.push({ Precio: precio, Ubicacion: loc, Titulo: titulo, Atributos: attrs, Link: link });
});

console.table(casas);

// Generar CSV con soporte para tildes (UTF-8)
let csvContent = "\uFEFFPrecio;Ubicacion;Titulo;Atributos;Link\n" 
    + casas.map(c => `"${c.Precio}";"${c.Ubicacion}";"${c.Titulo}";"${c.Atributos}";"${c.Link}"`).join("\n");

let blob = new Blob([csvContent], { type: 'text/csv;charset=utf-8;' });
let url = URL.createObjectURL(blob);
let downloadLink = document.createElement("a");
downloadLink.setAttribute("href", url);
downloadLink.setAttribute("download", "datos_mercadolibre.csv");
document.body.appendChild(downloadLink);
downloadLink.click();
document.body.removeChild(downloadLink);
